## Suggested Website

- If you have a public demo, place it in the GitHub repo "Website" field.

- Upload `docs/images/demo-console.png` as the repository social preview image.

## Release Checklist

1. Create tag `v0.1.0`
2. Use [RELEASE_v0.1.0.md](RELEASE_v0.1.0.md) as the initial release body
3. Pin the repository if it is one of your main projects
4. Add the About text and Topics on the GitHub repo home page
5. Share one short demo post with a real use case instead of a generic "open source project" announcement
